🎬 Splash / Welcome Screen

  • ▶ Animated splash screen shown on every app open, ~2 seconds
  • ▶ Display app logo + property branding (logo/colors configurable by property manager)
  • ▶ Subtle animation — fade-in, scale, or Lottie animation (see references: Four Seasons app, Centara Concierge, STAY hotel app — all use branded splash screens)
  • ▶ Transitions smoothly into the home screen
  • ▶ First-time users: splash leads to onboarding/document upload instead of home

🏠 Home Screen Breakdown

  • ▶ Welcome banner: dynamic greeting (good morning/afternoon/evening) + guest name + property name + "Day X of Y"
  • ▶ "Your Stay Info" button: opens a page with property address, wifi password, house rules, check-in/out times, emergency contacts, property photos
  • ▶ 2x3 category grid — 6 categories, all configurable by property manager:
  • ▶ • Food & Drinks — meals, snacks, wine & spirits, groceries, coffee, cocktails
  • ▶ • Experiences — boat trips, cooking classes, yoga, guided tours, excursions
  • ▶ • Discover — curated beaches, restaurants, hidden gems
  • ▶ • Wellness — massage, spa, personal trainer
  • ▶ • Transport — airport transfer, car rental, scooter
  • ▶ • Add-ons — extra towels, baby cot, BBQ kit, late checkout
  • ▶ Each category opens a sub-page with items to browse & add to cart
  • ▶ "Today's Specials" — horizontal scroll of promoted items (linked to push notifications)
  • ▶ Top icons: Account (top-left) opens profile/documents/order history. Notification bell (top-right) with unread badge
  • ▶ Floating chat button — bottom-right, always visible. Opens AI assistant + "Contact Support" escalation
  • ▶ No bottom navigation bar — home is the single main screen, everything accessible from grid/icons
App Features & Requirements

🔒 Access & Onboarding

  • ▶ App access granted from date A to date B (set by property manager per booking)
  • ▶ On first login, guest uploads ID/passport photo + fills personal info directly in the app
  • ▶ After document upload, guest lands on home screen with full access
  • ▶ After date B, app access is revoked (read-only or locked)

🛒 Orders & Payments

  • ▶ NO payment integration — no Stripe, no in-app payments
  • ▶ Guest adds items throughout their stay — these appear as a list of expenses inside the Account page
  • ▶ Order list shows running total, item details, quantities, status (pending/confirmed/fulfilled)
  • ▶ Payment happens at checkout (in person to property manager)
  • ▶ Property manager confirms/fulfills orders from the webapp

💬 Chat System

  • ▶ AI Assistant as first line of support — answers common questions (hours, directions, recommendations)
  • ▶ "Contact Support" button — escalates to human (property manager)
  • ▶ Messaging bridge: guest stays in-app, property manager receives on WhatsApp or Telegram
  • ▶ Evaluate: WhatsApp Business API vs Telegram Bot API (Telegram is cheaper & easier to integrate)
  • ▶ Chat history preserved for the duration of the stay

🔔 Push Notifications

  • ▶ Programmable by property manager from the webapp
  • ▶ Can include items/specials (e.g. "Tonight: fresh fish BBQ! Tap to order")
  • ▶ Tapping the notification deep-links to the item in the app
  • ▶ Schedule notifications in advance (e.g. every Tuesday at 5 PM)
  • ▶ Target: all current guests or specific bookings

👤 Account / Profile (top-left icon)

  • ▶ Guest name, email, phone
  • ▶ Uploaded documents (ID/passport)
  • ▶ Current booking details
  • ▶ Expenses / order list — running total of all items added during the stay, with status per item
  • ▶ Language preference

💻 Property Manager Webapp

  • ▶ Create & manage properties — name, address, photos, description, house rules, wifi info
  • ▶ Configure categories & services — create/rename/reorder categories, add items with name/description/photo/price
  • ▶ Manage bookings — set guest access dates (A to B), view guest documents
  • ▶ Order management — view incoming orders, confirm/decline, mark as fulfilled
  • ▶ Push notifications — compose, schedule, target guests
  • ▶ Chat overview — view AI conversations, respond to escalated chats
  • ▶ Specials / promotions — feature items on home screen + link to push notifications
Technology Recommendation

Option 1 — React Native + Expo (Recommended)

  • ▶ One codebase for iOS + Android
  • ▶ JavaScript/TypeScript — largest developer pool, easiest to hire
  • ▶ Expo handles builds, OTA updates, push notifications out of the box
  • ▶ Huge ecosystem of libraries for chat, image upload, navigation

Option 2 — Flutter

  • ▶ One codebase for iOS + Android
  • ▶ Excellent UI performance, beautiful animations out of the box
  • ▶ Dart language — smaller developer pool but growing fast
  • ▶ Good alternative if team already has Flutter experience

🛠️ Shared Stack (Both Options)

  • ▶ Backend: Supabase (Postgres + Realtime + Auth + Storage) or Firebase
  • ▶ AI Chat: Claude API or OpenAI API for the in-app assistant
  • ▶ Messaging bridge: Telegram Bot API (free, simpler) or WhatsApp Business API (paid, more familiar)
  • ▶ Push notifications: Firebase Cloud Messaging (FCM) — works for both iOS & Android
  • ▶ Property Manager Webapp: Next.js or similar, connected to same backend
